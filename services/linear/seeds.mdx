## Overview

Seed files define the initial state of a Linear workspace. They're JSON files that specify teams, issues, users, workflows, and more.

## Quick Reference: linear_expanded

## Seed File Location

Seeds are stored in `examples/linear/seeds/`:

```
examples/
└── linear/
    └── seeds/
        ├── linear_default.json
        ├── linear_expanded.json
        └── your_custom_seed.json
```

## Schema

### Organization

```json
{
  "organizations": [
    {
      "id": "org-uuid",
      "name": "Acme Corp",
      "urlKey": "acme"
    }
  ]
}
```

### Teams

```json
{
  "teams": [
    {
      "id": "team-uuid",
      "name": "Engineering",
      "key": "ENG",
      "description": "Engineering team",
      "organizationId": "org-uuid",
      "issueCount": 0
    }
  ]
}
```

### Users

```json
{
  "users": [
    {
      "id": "user-uuid",
      "name": "Agent Bot",
      "email": "agent@example.com",
      "displayName": "Agent Bot",
      "active": true,
      "organizationId": "org-uuid"
    }
  ]
}
```

### Team Memberships

```json
{
  "team_memberships": [
    {
      "id": "membership-uuid",
      "userId": "user-uuid",
      "teamId": "team-uuid"
    }
  ]
}
```

### Workflow States

```json
{
  "workflow_states": [
    {
      "id": "state-backlog-uuid",
      "name": "Backlog",
      "type": "backlog",
      "position": 0,
      "teamId": "team-uuid"
    },
    {
      "id": "state-inprogress-uuid",
      "name": "In Progress",
      "type": "started",
      "position": 1,
      "teamId": "team-uuid"
    },
    {
      "id": "state-done-uuid",
      "name": "Done",
      "type": "completed",
      "position": 2,
      "teamId": "team-uuid"
    }
  ]
}
```

| Type | Description |
|------|-------------|
| `backlog` | Backlog/Triage state |
| `unstarted` | Not started |
| `started` | In progress |
| `completed` | Done |
| `canceled` | Canceled |

### Issues

```json
{
  "issues": [
    {
      "id": "issue-uuid",
      "title": "Fix login bug",
      "description": "Users can't log in with SSO",
      "identifier": "ENG-1",
      "number": 1,
      "priority": 2,
      "stateId": "state-backlog-uuid",
      "teamId": "team-uuid",
      "assigneeId": "user-uuid",
      "creatorId": "user-uuid"
    }
  ]
}
```

| Priority | Label |
|----------|-------|
| 0 | No priority |
| 1 | Urgent |
| 2 | High |
| 3 | Medium |
| 4 | Low |

### Labels

```json
{
  "issue_labels": [
    {
      "id": "label-uuid",
      "name": "bug",
      "color": "#EB5757",
      "teamId": "team-uuid"
    }
  ]
}
```

### Issue-Label Associations

```json
{
  "issue_label_issue_association": [
    {
      "issueId": "issue-uuid",
      "labelId": "label-uuid"
    }
  ]
}
```

### Comments

```json
{
  "comments": [
    {
      "id": "comment-uuid",
      "body": "I'm looking into this now.",
      "issueId": "issue-uuid",
      "userId": "user-uuid"
    }
  ]
}
```

### Projects

```json
{
  "projects": [
    {
      "id": "project-uuid",
      "name": "Q1 Roadmap",
      "state": "started",
      "progress": 0.5
    }
  ]
}
```

## Creating a Custom Template

1. Create your seed file in `examples/linear/seeds/my_seed.json`

2. Register the template (during seed script execution)

3. Use your template:

```python
env = client.init_env(
    templateService="linear",
    templateName="my_custom_template",
    impersonateEmail="agent@example.com"
)
```

## Tips

<Tip>
**UUIDs**: Use proper UUIDs for IDs. You can generate them with Python's `uuid.uuid4()`.
</Tip>

<Tip>
**Issue Numbers**: Keep `identifier` and `number` consistent with team `key`. For team "ENG", issue number 1 should have identifier "ENG-1".
</Tip>

<Warning>
**Foreign Keys**: Ensure all referenced IDs exist. An issue's `teamId`, `stateId`, and `assigneeId` must reference valid entities.
</Warning>
